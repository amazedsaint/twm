Transactional World Loop

A proposer emits a scalar program, a projector makes a typed candidate, the hard verifier rejects the first trace with a residual, the learner repairs the next trace, and only the accepted replayable transition commits.

set(0) -> reject(delta=5)
receipt residual -> learn add(5)
set(0), add(5) -> commit
